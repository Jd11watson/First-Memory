# Xcode Project Setup Guide

The Swift source files are ready. Follow these steps to create the Xcode project and wire everything together.

## 1. Create Xcode Project

1. Open Xcode → File → New → Project
2. Choose **App** (iOS)
3. Settings:
   - Product Name: `LifeTracker`
   - Bundle Identifier: `com.yourname.lifetracker` (remember this — used in App Groups)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Storage: **SwiftData** (check this box)
4. Save to: `life-tracker/ios/`

## 2. Add Source Files

Delete the default generated files (ContentView.swift, [AppName]App.swift) and add all files from `LifeTracker/`:
- Drag the entire `LifeTracker/` folder into Xcode
- Choose "Create groups" not "Create folder references"

## 3. Configure Capabilities

In Xcode → Your Target → Signing & Capabilities → "+ Capability":

| Capability | Required |
|---|---|
| HealthKit | YES |
| CloudKit | YES (for iCloud sync to dashboard) |
| Push Notifications | YES |
| Background Modes | YES — check: Background fetch, Remote notifications |
| App Groups | YES — add `group.com.yourname.lifetracker` |
| Family Controls | YES — for DeviceActivity (screen time / pickups) |

## 4. Info.plist Keys

Add these keys to your Info.plist:

```xml
<key>NSHealthShareUsageDescription</key>
<string>Life Tracker reads your sleep, heart rate, and workout data to surface patterns and insights.</string>

<key>NSHealthUpdateUsageDescription</key>
<string>Life Tracker does not write to Health. This key is required by the framework.</string>

<key>NSLocationWhenInUseUsageDescription</key>
<string>Life Tracker uses your location to detect when you visit restaurants and to pull local weather data.</string>

<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>Life Tracker uses significant location changes to detect venue visits without draining your battery.</string>

<key>UIBackgroundModes</key>
<array>
    <string>fetch</string>
    <string>remote-notification</string>
</array>

<key>BGTaskSchedulerPermittedIdentifiers</key>
<array>
    <string>com.lifetracker.healthsync</string>
</array>
```

## 5. Widget Extension

1. File → New → Target → Widget Extension
2. Name: `LifeTrackerWidgets`
3. Add to target: the widget files from `Widgets/LockScreen/` and `Widgets/EveningCheckIn/`
4. Add the widget extension to the same App Group: `group.com.yourname.lifetracker`
5. In widget target → Build Phases → Link Binary with Libraries: add `WidgetKit.framework` and `ActivityKit.framework`

## 6. CloudKit Container

1. In your main target → Signing & Capabilities → CloudKit
2. Create a new container: `iCloud.com.yourname.lifetracker`
3. In CloudKit Dashboard (developer.apple.com), the schema will be auto-generated by SwiftData

## 7. Family Controls Entitlement

The Family Controls entitlement requires a special entitlement file:

Create `LifeTracker.entitlements`:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.developer.family-controls</key>
    <true/>
    <key>com.apple.developer.healthkit</key>
    <true/>
    <key>com.apple.developer.icloud-services</key>
    <array>
        <string>CloudKit</string>
    </array>
    <key>com.apple.developer.icloud-container-identifiers</key>
    <array>
        <string>iCloud.com.yourname.lifetracker</string>
    </array>
    <key>com.apple.security.application-groups</key>
    <array>
        <string>group.com.yourname.lifetracker</string>
    </array>
    <key>aps-environment</key>
    <string>development</string>
</dict>
</plist>
```

## 8. Distribution

For personal use, distribute via **TestFlight**:
- No App Store review required
- Family Controls entitlement works for personal testing
- Build → Product → Archive → Distribute → TestFlight

## 9. Build & Test

```bash
# Build from CLI (optional)
xcodebuild -project LifeTracker.xcodeproj -scheme LifeTracker -destination 'platform=iOS Simulator,name=iPhone 15 Pro'
```
